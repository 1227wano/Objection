### [TIL] 프로젝트 기획 구체화

1. 초기 데이터 탐색 및 프론트엔드 렌더링 고민

    주제 탐색: 프로젝트 초기 아이디어로 '전력 공급량 데이터'를 시각화하는 방안을 검토했다.

    기술적 검토: 프론트엔드 단에서 지도 위에 데이터를 어떻게 효율적으로 그릴 수 있을지 고민했다. 특히 공간 데이터를 다루기 위해 API 응답이 GeoJSON 포맷으로 들어오는지 파악하고, 폴리곤이나 마커를 렌더링하는 전략을 구상했다.

2. 기획 확정: 전기차 바로충전 서비스

    팀원들과의 회의를 거쳐, 최종적으로 '전기차 충전 가능 예측 서비스' 기획안으로 합의 및 확정했다.

    Hadoop과 Spark 등을 이용해 과거 이력(1,220만 건) 패턴을 분석하고 실시간 교통/충전 상태를 결합하는 고도화된 로직이다.
사용자가 목적지에 도착하는 미래 시점(t)에 빈 충전기가 있을 도착 시점 확률을 계산하여 제공한다.

3. 프론트엔드 UI/UX 전략 고민

    UX 딜레마 해결 (직관성 vs 재미):

    문제: 데이터를 재미있고 화려하게 보여주는 것도 좋지만, 배터리가 부족해 마음이 급한 운전자에게 너무 많은 시각적 자극은 인지 부하를 유발할 수 있다.

    해결 방안:
    운전/주행 중 엔 직관적으로, 정차나 탐색중에는 여러 시각적 요소를 활용해서 데이터를 재미있게 제공하고자 하면 어떨까 생각했다.
